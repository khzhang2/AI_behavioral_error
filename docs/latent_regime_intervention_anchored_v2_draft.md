# Optima Intervention-Anchored Latent-Regime Experiment V2: Design Outline

This draft defines a second-stage methodological experiment whose primary purpose is not to pre-classify artificial intelligence (AI) response errors into a fixed taxonomy, but to identify whether the observed variation in AI survey responses can be decomposed into exact-repeat randomness, systematic intervention sensitivity, and a small number of latent response regimes. The central design principle is therefore sequential identification. The experiment first establishes a stochastic baseline under exact repetition, then evaluates whether controlled perturbations generate response shifts that exceed that baseline, and only then estimates a Scale-Adjusted Latent Class Choice Model (SALCM) on the structured differences that remain. Human data are used as a benchmark for behavioral distortion rather than as a direct participant in the latent-regime model.

## Conceptual Logic

The experimental object is a respondent block rather than a single task. A block is defined by model identity, prompt family, prompt arm, sampled persona, task panel template, and one independent run repeat. The same block template must be executed repeatedly in order to identify within-model instability under identical stimuli. This repeated-block design is essential because a latent regime should be interpreted as a stable hidden response state only after purely stochastic variation has been separated from structured intervention effects.

The identification logic proceeds in four steps.

1. Exact-repeat randomness is measured first. For a fixed block template and fixed task, the experiment repeats the same session independently and computes the exact-repeat flip rate and response entropy. This produces a stochastic envelope for each task type.
2. Intervention sensitivity is then measured by introducing manipulations that, by design, should not alter the underlying normative utility. These manipulations include semantic paraphrase, label masking, and order randomization. Dominance and monotonicity tasks are retained as direct diagnostics of trade-off fidelity rather than as utility-equivalent perturbations.
3. Only the systematic variation that exceeds the randomness envelope is passed to the latent-regime stage. At that stage, SALCM is used to determine whether the structured differences can be summarized by a small number of preference classes and scale classes.
4. The recovered regimes are interpreted relative to a human benchmark. The experiment therefore does not ask whether AI is merely different from humans in aggregate. It asks whether particular regimes produce identifiable behavioral distortions relative to the human benchmark in mode shares, values of time or values of waiting, subgroup patterns, and diagnostic compliance.

This sequencing is the substantive methodological contribution. The experiment is not a conventional latent-class application, because latent classes are not used as a first response to any observed heterogeneity. They are introduced only after the null of pure stochastic instability has been challenged by intervention evidence.

## Hypotheses

The hypotheses should be defined in a way that matches the sequential logic of the design.

### H0: Pure stochastic-instability hypothesis

For any intervention pair that is designed to preserve normative utility, the observed response gap does not exceed the stochastic envelope generated by exact-repeat randomness. Let \(R^{\mathrm{rep}}_{bt}\) denote the exact-repeat flip rate for block template \(b\) and task \(t\), and let \(G_{bt}(a,a')\) denote the intervention gap between two utility-equivalent task realizations \(a\) and \(a'\). The null hypothesis is that, for a pre-specified tolerance factor \(\kappa\),

\[
G_{bt}(a,a') \le \kappa R^{\mathrm{rep}}_{bt}.
\]

If this null is rejected systematically, the experiment supports the claim that the observed shifts are not reducible to within-model randomness alone.

### H1: Latent-regime hypothesis

The intervention effects that exceed the stochastic envelope are not unstructured residual variation. Instead, they can be summarized by a small number of latent response regimes whose differences are expressed through both utility parameters and scale parameters. Under this hypothesis, a SALCM with a limited number of preference classes and scale classes should provide a substantively interpretable compression of the joint signatures formed by choices, intervention gaps, confidence, and diagnostic compliance.

### H2: Membership-structure hypothesis

Observed exogenous conditions change the probability that a block belongs to a given latent regime. These conditions include model identity, prompt arm, prompt family, task complexity, and any other experimental burden measures retained in the design. This hypothesis does not claim full causal source identification, but it does require that class-membership probabilities vary systematically with experimentally assigned or otherwise exogenous block-level covariates.

## Recommended Experimental Architecture

The design should retain the current Optima domain and the current human benchmark, but it should expand the repeated-task architecture to support exact-repeat identification.

Each respondent block should contain six psychometric questions and a panel of choice tasks. The task panel should be generated from a block template that is held fixed across independent repeats. A practical V2 design would contain the following elements.

- A set of core empirical tasks drawn from the Optima scenario bank.
- Semantic-paraphrase twins that alter wording without changing the displayed attribute values.
- Label-mask twins that replace mode labels with neutral alternatives such as Option A, Option B, and Option C.
- Order-randomization twins that preserve attributes and labels but alter presentation order.
- Monotonicity diagnostics that worsen one alternative on salient attributes.
- Dominance diagnostics that create a clearly dominated option.

The exact-repeat stage requires that each block template be run multiple times as independent sessions. The natural minimum is three repeats per block template, because this allows pairwise flip-rate estimation without making the survey volume unmanageable.

## Estimation Sequence

The estimation sequence should remain disciplined.

First, estimate the human benchmark Multinomial Logit (MNL) model on the retained Optima human data. This step defines the normative benchmark and should not be mixed with the repeated-task AI panel.

Second, compute exact-repeat randomness and intervention gaps directly from the repeated-task AI data before estimating any latent-class model. The key statistical object is the excess intervention gap, defined as the intervention gap net of the stochastic envelope.

Third, estimate a pooled AI panel MNL as the non-latent baseline on the repeated-task AI data. This baseline clarifies what is gained by allowing latent preference and scale heterogeneity.

Fourth, estimate a pooled AI SALCM only on the structured repeated-task data. The preferred specification should retain separate preference classes and scale classes, because the central methodological question is whether the excess intervention sensitivity is better understood as preference distortion, consistency distortion, or both.

Fifth, compute regime-level distortion summaries relative to the human benchmark and respondent-level posterior distortion scores using posterior class probabilities.

## Interpretation Rule

Recovered classes should not be pre-labeled as cost-neglect regimes, label-bias regimes, or random-answer regimes. Instead, each class should first be described in terms of its empirical signature: exact-repeat stability, intervention sensitivity, monotonicity compliance, dominance compliance, confidence profile, and distance from the human benchmark. Only after this empirical profile has been established should the class be interpreted substantively. This preserves the methodological distinction between latent statistical structure and causal or psychological naming.

## Practical Implication for V2 Implementation

The current V1 line already contains the core ingredients of repeated tasks, diagnostic tasks, structured responses, and SALCM. The main V2 additions should therefore be limited and targeted. The data-preparation stage must explicitly create reusable block templates and independent run repeats. The collection stage must preserve the distinction between block templates and realized sessions. The analysis stage must compute exact-repeat randomness and intervention gaps before SALCM is estimated. The final summary stage must report whether intervention effects exceed the stochastic envelope, whether SALCM meaningfully compresses the remaining structured variation, and how each recovered regime distorts behavior relative to the human benchmark.
