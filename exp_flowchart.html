Optima Intervention-Regime 实验流程图（更新版）

这版把一次真实实验的 prepare 与 collection 流程一并画进来，包含 persona 构建、 choice card 生成、`block_assignments.csv`、`panel_tasks.csv`、原始 AI 输出，以及后续估计与比较。 节点支持鼠标悬停显示解释，并继续保留按阶段的浅色透明分组框。

打开页面后直接把鼠标移到图中任意节点查看说明；离开节点则自动消失。
`task` 单道题。可以是态度题、choice card，或 diagnostic/probe 题。
`task bundle` 一组任务内容的集合，说明“这一轮问卷里有什么题”。
`block template` 可复用的问卷会话模板：`task bundle + ordering + probe design + output schema`。
`respondent block` 把 `block template` 放到固定实验条件下形成的实验单元。
`run` 对一个 `respondent block` 的一次完整实际执行，也就是“完整喂完一轮问卷”。
`persona` 从 human profile bank 抽取并转写成 AI 受访者背景的文本画像，用于 system prompt。
`choice card` 单张 stated-preference 选项卡，一行任务记录，包含 A/B/C 选项及其属性。
`support set` 当前实现中的 persona 支持集是经验支持集：只使用人类数据里真实出现过的完整 profile rows，不做属性全枚举。

问答次数计算例子

下面用刚才那次 small-scale smoke test 说明一次实验总共会产生多少次 AI 问答请求。

参数 当前例子 含义
n_block_templates_per_model 10 每个模型生成 10 个问卷块模板。
n_repeats_per_template 2 每个模板做 2 次 exact repeat。
n_attitudes 6 每个 run 里有 6 道态度题。
n_core_tasks 6 每个 run 里有 6 张核心 choice cards。
n_paraphrase_twins 2 每个 run 里有 2 张 paraphrase cards。
n_label_mask_twins 2 每个 run 里有 2 张 label-mask cards。
n_order_twins 2 每个 run 里有 2 张 order-randomization cards。
n_monotonicity_tasks 2 每个 run 里有 2 张 monotonicity cards。
n_dominance_tasks 2 每个 run 里有 2 张 dominance cards。

所以每个 run 的 task cards 总数是： n_core_tasks + n_paraphrase_twins + n_label_mask_twins + n_order_twins + n_monotonicity_tasks + n_dominance_tasks = 6 + 2 + 2 + 2 + 2 + 2 = 16

每个 run 的 AI 问答请求总数是： 1 grounding + 6 attitudes + 16 tasks = 23

总 runs 数是： n_block_templates_per_model × n_repeats_per_template = 10 × 2 = 20

所以整次实验的总问答请求数是： 20 × 23 = 460

一般公式可以写成： 总请求数 = n_block_templates_per_model × n_repeats_per_template × (1 + n_attitudes + 总 task cards)
